## 执行流程

### 第1步：读取参考文档
- 读取 `references/architecture-patterns.md` 了解工业软件常用架构模式
- 根据需求按需读取 `references/open-source-libraries.md` 了解候选开源库
- 涉及设备互联时读取 `references/device-connectivity.md`
- 涉及安全合规时读取 `references/security-compliance.md`

### 第2步：需求分析
- 明确系统规模（单体 / 分布式 / 边缘计算）
- 确定关键质量属性（性能 / 可靠性 / 可扩展性 / 安全性）
- 识别技术约束（语言、平台、License、团队技能）

### 第3步：架构模式选择

| 系统规模 | 推荐架构 | 典型场景 |
|----------|----------|----------|
| 单机 CAD/CAE 工具 | 分层 + 插件化 | FreeCAD 式桌面应用 |
| 工业物联网平台 | 微服务 + 边缘计算 | 数据采集 + 监控 |
| 工业仿真系统 | 分层 + 管道模式 | 前处理 → 求解 → 后处理 |
| 嵌入式 HMI | MVC + 事件驱动 | 设备监控面板 |

### 第4步：输出设计文档

包含以下内容：

1. **架构决策记录**（ADR）
   - 决策：选择了什么
   - 背景：为什么需要做决策
   - 候选方案：考虑了哪些选项
   - 决策理由：为什么选这个

2. **模块划分**
   - 核心模块及其职责
   - 模块间依赖关系
   - 接口定义（C++ 头文件或 Python Protocol）

3. **技术选型对比表**

4. **关键设计决策**
   - 数据一致性策略
   - 服务间通信协议
   - 容错与恢复方案

5. **风险评估**
   - 技术风险
   - 性能瓶颈
   - License 风险

## 架构设计原则（工业软件专用）
- 几何建模、可视化、数据管理三层严格分离
- 业务逻辑不直接依赖具体库类型（OCCT/OSG/VTK），通过抽象接口
- 插件通过接口注册，支持动态加载（参考 FreeCAD Workbench）
- 大型运算异步执行，UI 保持响应
- 工业数据安全设计内置，而非后期补充
